## 一、問題陳述

### 「今晚吃什麼？」是一個每天都在上演的群體決策困境

```
群組對話：
A：今晚吃什麼？
B：都可以
C：隨便
D：你決定
A：......
```

**這不是溝通問題，而是決策機制的問題。**

每個人在餐廳議題上「不表態」，背後有一個理性的動機：
- 提出建議 = 承擔被嫌的風險
- 大家認同你的選擇 = 以後每次都找你決定
- 最安全的策略：沉默，讓別人先說

這種現象在心理學上稱為**「責任分散效應」**（Diffusion of Responsibility）——在群體情境中，個人傾向讓責任由他人分擔，導致集體行動障礙。

## 二、市場現況分析

| 現有解法 | 問題所在 |
|---------|---------|
| Google Maps | 200 間選項 → 更嚴重的選擇困難 |
| 美食評比平台 | 排行榜代表大眾，不代表你們今晚的心情 |
| 轉輪盤 App | 完全隨機，不考慮任何人的偏好 |
| LINE 群組投票 | 仍需有人先想選項，且公開投票會引發從眾 |

**現有解法的共同盲點：**
> 所有工具都在增加資訊或幫助你決定，但沒有一個工具能讓「沒有人需要負責」這件事真正發生。

## 三、解決方案：佛系揪團

### 核心理念
> **你負責聚，我負責選。**
> No Pressure Decision — 讓命運替你們做決定。

### 三步完成決策

```
Step 1  選擇大類別
        🍖 正餐 / 🍢 小吃宵夜 / 🍰 甜點下午茶

            ↓

Step 2  Pass & Play 盲測投票
        把手機遞給下一個人
        每人獨立對 8 個品項按 ❤️ 或 ✖️
        看不到別人的選擇（消除從眾效應）

            ↓

Step 3  3 ... 2 ... 1 ... 🎉
        彩色紙花開獎
        系統宣告命運的選擇
        一鍵開啟 Google Maps 找附近的店
```

## 四、核心設計決策

### 決策 1：為什麼用盲測，不用公開投票？

公開投票（如 LINE 投票）的問題：先投的人影響後投的人，最終結果反映的是「社會規範」而不是「真實偏好」。

盲測讓每個人**面對自己的胃口**，而不是面對朋友的眼光。最終的計票結果因此更真實地代表群體共識。

### 決策 2：為什麼要有開獎儀式？

如果直接跳出結果，這只是一個工具。

有了 3-2-1 倒數 + 彩色紙花，它變成一個**共同體驗的時刻**。每個人都看著同一個畫面等待，然後一起歡呼或嘆氣。這個記憶點讓使用者願意再次使用，也容易在社交媒體上引發截圖分享。

### 決策 3：為什麼平手時用亂數，而不讓使用者再投一輪？

因為讓使用者「再做決定」違背了產品的核心承諾。「系統替你決定」必須被絕對貫徹，才能從根本上消除責任感壓力。

### 決策 4：為什麼用 Pass & Play，而不是多裝置聯線？

**MVP 的核心假設是降低參與門檻。**

如果需要每個人都安裝 App 或登入帳號，很多人會在這一步放棄。Pass & Play（傳一支手機輪流投票）讓零門檻成為可能：掃 QR Code 或點連結即可，連帳號都不需要。

## 五、技術架構

### 技術棧選擇

| 技術 | 選擇理由 |
|------|---------|
| **Blazor WebAssembly** | 純 C# 開發，不需後端，可部署為靜態網站 |
| **MudBlazor** | Material Design 元件，快速建構精緻介面 |
| **C# Dictionary** | 即時計票，輕量高效 |
| **Pure CSS Keyframes** | confetti 紙花、彈跳動畫，零外部依賴 |
| **Google Maps URL Scheme** | 不需 API Key，直接跳轉，零成本整合 |
| **GitHub Pages + Actions** | 每次 push 自動部署，月費 $0 |

### 架構優勢
```
傳統架構：前端 → 後端 API → 資料庫
          月費 $50–$500 + 維護負擔

佛系揪團：Blazor WASM → 瀏覽器本地執行
          月費：$0
          資料：永遠不離開使用者的裝置（隱私保護）
```

## 八、結論

「佛系揪團」解決的不是「哪間餐廳好吃」的問題，而是「**誰來做決定**」的問題。

透過盲測機制消除從眾、透過亂數仲裁消滅責任、透過開獎儀式創造共同記憶，我們把一個每天讓人頭痛的日常困境，變成了一場快樂的命運抽籤。

這個產品的價值不在功能的複雜度，而在於**它精準擊中了人性中最真實的痛點**：沒有人想負責，所以我們讓系統來負責。
